# Contributing to AI-Powered Intrusion Detection System

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Follow ethical security practices
- Report security issues responsibly

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/AI-Intrusion-Detection-System.git`
3. Create a feature branch: `git checkout -b feature/YourFeature`
4. Install dependencies: `pip install -r requirements.txt`

## Development Process

### Code Style
- Follow PEP 8 guidelines
- Use type hints
- Write docstrings for all functions
- Keep lines under 100 characters

### Testing
- Write tests for new features
- Run tests: `pytest tests/ -v`
- Maintain >80% code coverage
- Test edge cases and error scenarios

### Commit Messages
- Use clear, descriptive commit messages
- Format: `[TYPE] Description` where TYPE is:
  - `feat`: New feature
  - `fix`: Bug fix
  - `docs`: Documentation
  - `test`: Tests
  - `refactor`: Code refactoring

### Pull Request Process

1. Update documentation
2. Add tests for new functionality
3. Update CHANGELOG.md
4. Run tests and linting
5. Submit PR with clear description
6. Address review feedback
7. Merge after approval

## Areas for Contribution

- **Features**: New detection methods, alerts, response actions
- **Models**: Additional ML algorithms, ensemble techniques
- **Performance**: Optimization, caching, GPU acceleration
- **Documentation**: Guides, tutorials, examples
- **Testing**: More comprehensive test coverage
- **Security**: Vulnerability fixes, security enhancements

## Reporting Issues

When reporting bugs:
1. Use descriptive title
2. Describe expected vs actual behavior
3. Provide steps to reproduce
4. Include system/environment info
5. Attach relevant logs/screenshots

## Feature Requests

1. Describe the feature
2. Explain use case and benefits
3. Provide implementation suggestions if available
4. Link related issues/discussions

## Questions & Discussions

- Use GitHub Discussions for general questions
- Check existing issues before asking
- Be specific and provide context
